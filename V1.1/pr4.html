Abstract

This specification lists which classes and properties of the CIDOC Conceptual Reference Model are used in ArchOnto. This specification takes into account the CIDOC CRM standard in version 7.1.3, which is regarded as the Official ISO Correspondence.

Only the classes and properties essential for understanding and operating ArchOnto were considered. As CIDOC CRM is developed by an external institution, it was necessary to identify the core elements required for its correct application within ArchOnto.

This specification also allows tracking the use of these elements over time.

While any CIDOC CRM class can be used in ArchOnto, the ones presented here are those deemed fundamental at the moment of the publication of this version of ArchOnto.

0100 Item
Any item or product for which specific information needs to be retrieved.
An Item (article, product, item, goods item, object) is the smallest unit managed as an entity within the Prisma® 4 system. It represents any material, spare part, consumable or element acquired and stored in the supplies warehouse, which can be used in construction and maintenance work, by both internal and external operators.

0102 gtin
This property .
In particular it captures the characterisation of the item itself, its internal structures, appearance etc.
Like property P2 has type (is type of), this property is a consequence of the restricted focus of the CRM. The aim is not to capture, in a structured form, everything that can be said about an item; indeed, the CRM formalism is not regarded as sufficient to express everything that can be said. Good practice requires use of distinct note fields for different aspects of a characterisation. The P3.1 has type property of P3 has note allows differentiation of specific notes, e.g. “construction”, “decoration” etc.
An item may have many notes, but a note is attached to a specific item.
